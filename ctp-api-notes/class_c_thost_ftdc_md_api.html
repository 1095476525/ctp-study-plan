static CThostFtdcMdApi* CThostFtdcMdApi::CreateFtdcMdApi ( const char *  pszFlowPath = "",
const bool  bIsUsingUdp = false,
const bool  bIsMulticast = false 
)
static

This function creates an CThostFtdcMdApi instance.

Parameters
[in]pszFlowPathA constant string indicates a file directory which is used to store session information. If not specified, the current file directory will be used. The specified file directory must be pre-created, and the path should be '/' terminated(e.g. "./tmp/"), api will create some temp files in the folder, so make sure the progress has the appropriate access to the folder.
[in]bIsUsingUdpWhether to use UDP, should always be false.
[in]bIsMulticastWhether to use multicast, should always be false.
Returns
A pointer to an instance of CThostFtdcMdApi.
Sample code
CThostFtdcMdApi *pMdApi = CThostFtdcMdApi::CreateFtdcMdApi("md/");
Warning
This is the only way to get the instance of CThostFtdcMdApi, never use 'new' keyword or malloc()/calloc() to do it. And never use 'delete' keyword, free() or realloc() to release it, call CThostFtdcMdApi::Release() instead.
If two or more instances need to run concurrently, the pszFlowPath must be different between each two instances. CThostFtdcMdApi instance and CThostFtdcTraderApi instance also need to use different flow paths. Different login user also need to use different flow paths. It is suggested to use "md_<userid>_<threadid>" as the folder name.
If pszFlowPath is specified, the path must terninated with '/' and the folder must be pre-created.
See also
CThostFtdcMdApi::Release()

◆ RegisterFront()

virtual void CThostFtdcMdApi::RegisterFront ( char *  pszFrontAddress)
pure virtual

The client application uses this function to register the front address of the server.

The function can be invocated more than one time to register more front addresses, and the API will selected one until the connection is created successfully.

Parameters
[in]pszFrontAddressPointer of the structure for the front address of the CTP server. The address format is just like: "protocol://host:port", for example, "tcp://127.0.0.1:17001", "tcp" means the communication protocol, "127.0.0.1" identifies the host name((or ip address) of server, "17001" identifies the server port.
Sample code
pMdApi->RegisterFront("tcp://127.0.0.1:17001");
See also
CThostFtdcMdApi::CreateFtdcMdApi()

◆ Release()

virtual void CThostFtdcMdApi::Release ( )
pure virtual

Use this function to destroy the CThostFtdcMdApi instance and release the resources, and wake up all the threads that blocked on CThostFtdcMdApi::Join() of the same instance.

Sample code
if (pMdApi != NULL) {
doSleep(100); // Add a delay(100ms) between Init() and Release().
pMdApi->RegisterSpi(NULL);
pMdApi->Release();
pMdApi = NULL;
}
Warning
This function cannot be called in any callback functions of the CThostFtdcMdApi instance.
This function cannot be called till the thread of the instance is really started. The call of Release() will be blocked forever if the thread of the instance is not ready, and the resources cannot be destroied/released correctly. The following code will make the current thread hang on the call of Release():
pMdApi->Init();
pMdApi->RegisterSpi(NULL);
pMdApi->Release(); // Got blocked.
pMdApi = NULL;
It is suggested to add a delay before Release():
pMdApi->Init();
doSleep(100); // Add a delay(100ms) between Init() and Release().
pMdApi->RegisterSpi(NULL);
pMdApi->Release(); // Will return soon.
pMdApi = NULL;
We've changed the delay duration to 50ms of the code above and tested thousands of times and never got blocked, users are suggested to use 100ms as the delay since the runtime environment is inconsistent between different machines and different run, 100ms should be safe enough for most of the cases.
The CThostFtdcMdApi instance should never be used once released.
See also
CThostFtdcMdApi::CreateFtdcMdApi(), CThostFtdcMdApi::Join()

The client application uses this function to send the login request to the server.

Parameters
[in]pReqUserLoginFieldA pointer to an CThostFtdcReqUserLoginField instance.
[in]nRequestIDThe request id.
Return values
0Success.
-1Network connection failure.
-2Exceeded the limit of max quantity of buffered(unhandled) requests.
-3Exceeded the limit of max requests per second.
Sample code
CThostFtdcReqUserLoginField field;
memset(&field, 0, sizeof(field));
strcpy(field.BrokerID, BROKER_ID);
strcpy(field.UserID, USER_ID);
strcpy(field.Password, USER_PASSWORD);
int rtnCode = mdApi->ReqUserLogin(&field, nextRequestID());
if (rtnCode != 0) {
Logger::info("[ERROR] [%s:%3d] Request failed: code=%d.", __FUNCTION__, __LINE__, rtnCode);
} else {
Logger::info("[INFO] [%s:%3d]: Requested to login: brokerID=%s, userID=%s, password=%s.", __FUNCTION__, __LINE__,
field.BrokerID, field.UserID, field.Password);
}
