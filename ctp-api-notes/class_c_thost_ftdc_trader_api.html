◆ CreateFtdcTraderApi()

static CThostFtdcTraderApi* CThostFtdcTraderApi::CreateFtdcTraderApi ( const char *  pszFlowPath = "")
static

This function creates an CThostFtdcTraderApi instance.

Parameters
[in]pszFlowPathA constant string indicates a file directory which is used to store session information. If not specified, the current file directory will be used. The specified file directory must be pre-created, and the path should be '/' terminated(e.g. "./tmp/"), api will create some temp files in the folder, so make sure the progress has the appropriate access to the folder.
Returns
A pointer to an instance of CThostFtdcTraderApi.
Sample code
CThostFtdcTraderApi *pTraderApi = CThostFtdcTraderApi::CreateFtdcTraderApi();
Warning
This is the only way to get the instance of CThostFtdcTraderApi, never use 'new' keyword or malloc()/calloc() to do it. And never use 'delete' keyword, free() or realloc() to release it, call CThostFtdcTraderApi::Release() instead.
If two or more instances need to run concurrently, the pszFlowPath must be different between each two instances. CThostFtdcMdApi instance and CThostFtdcTraderApi instance also need to use different flow paths. Different login user also need to use different flow paths. It is suggested to use "trade_<userid>_<threadid>" as the folder name.
If pszFlowPath is specified, the path must terninated with '/' and the folder must be pre-created.
See also
CThostFtdcTraderApi::Release()

◆ RegisterFront()

virtual void CThostFtdcTraderApi::RegisterFront ( char *  pszFrontAddress)
pure virtual

The client application uses this function to register the front address of the server.

The function can be invocated more than one time to register more front addresses, and the API will selected one until the connection is created successfully.

Parameters
[in]pszFrontAddressPointer of the structure for the front address of the CTP server. The address format is just like: "protocol://host:port", for example, "tcp://127.0.0.1:17001", "tcp" means the communication protocol, "127.0.0.1" identifies the host name((or ip address) of server, "17001" identifies the server port.
Sample code
pTraderApi->RegisterFront("tcp://127.0.0.1:17001");

pTraderApi->RegisterSpi(NULL);
pTraderApi->Release();
delete tradeSpi;
}

◆ Release()

virtual void CThostFtdcTraderApi::Release ( )
pure virtual

Use this function to destroy the CThostFtdcTraderApi instance and release the resources, and wake up all the threads that blocked on CThostFtdcTraderApi::Join() of the same instance.

Sample code
if (pTraderApi != NULL) {
doSleep(100); // Add a delay(100ms) between Init() and Release().
pTraderApi->RegisterSpi(NULL);
pTraderApi->Release();
pTraderApi = NULL;
}
Warning
This function cannot be called in any callback functions of the CThostFtdcTraderApi instance.
This function cannot be called till the thread of the instance is really started. The call of Release() will be blocked forever if the thread of the instance is not ready, and the resources cannot be destroied/released correctly. The following code will make the current thread hang on the call of Release():
pTraderApi->Init();
pTraderApi->RegisterSpi(NULL);
pTraderApi->Release(); // Got blocked.
pTraderApi = NULL;
It is suggested to add a delay before Release():
pTraderApi->Init();
doSleep(100); // Add a delay(100ms) between Init() and Release().
pTraderApi->RegisterSpi(NULL);
pTraderApi->Release(); // Will return soon.
pTraderApi = NULL;
We've changed the delay duration to 50ms of the code above and tested thousands of times and never got blocked, users are suggested to use 100ms as the delay since the runtime environment is inconsistent between different machines and different run, 100ms should be safe enough for most of the cases.
The CThostFtdcTraderApi instance should never be used once released.
See also
CThostFtdcTraderApi::CreateFtdcTraderApi(), CThostFtdcTraderApi::Join()

virtual int CThostFtdcTraderApi::ReqOrderAction ( CThostFtdcInputOrderActionField *  pInputOrderAction,
int  nRequestID 
)
pure virtual

Use this function to replace or cancel an order.

Parameters
[in]pInputOrderActionA pointer to an CThostFtdcInputOrderActionField instance.
[in]nRequestIDThe request id.
Return values
0Success.
-1Network connection failure.
-2Exceeded the limit of max quantity of buffered(unhandled) requests.
-3Exceeded the limit of max requests per second.
Sample code
CThostFtdcInputOrderActionField field;
memset(&field, 0, sizeof(field));
strcpy(field.BrokerID, BROKER_ID);
strcpy(field.InvestorID, INVESTOR_ID);
strcpy(field.UserID, USER_ID);
// Use FrontID+SessionID+OrderRef to locate the order:
field.FrontID = frontID; // FrontID can be got from CThostFtdcOrderField.FrontID
field.SessionID = sessionID; // SessionID can be got from CThostFtdcOrderField.SessionID
strcpy(field.OrderRef, orderRef);
// Or use OrderSysID to locate the order:
//strcpy(field.OrderSysID, "AAAAAAA");
// To cancel an order:
field.ActionFlag = THOST_FTDC_AF_Delete;
// Or to replace an order:
//field.ActionFlag = THOST_FTDC_AF_Modify;
//field.LimitPrice = price; // updated price
//field.VolumeChange = volume; // updated total volume
int rtnCode = tradeApi->ReqOrderAction(&field, nextRequestID());
if (rtnCode != 0) {
Logger::info("[ERROR] [%s:%3d]: Request failed: code=%d.", __FUNCTION__, __LINE__, rtnCode);
} else {
Logger::info("[INFO] [%s:%3d]: Requested to cancel order: orderRef=%s, frontID=%d, sessionID=%d.", __FUNCTION__, __LINE__,
field.OrderRef, field.FrontID, field.SessionID);
}
Note
Replace order is not supported for APEX and KSE and all the China future exchanges(except for HKE).
User can use 'FrontID + SessionID + OrderRef' or 'ExchangeID + OrderSysID' to locate an order. These id can be found from CThostFtdcOrderField with function CThostFtdcTraderSpi::OnRtnOrder() and CThostFtdcTraderSpi::OnRspQryOrder().
The request will firstly be checked by CTP server, if everyting is ok, it will be forwarded to the next hop(brokage or exchange).
The response from server of the request should be one of:
  • CThostFtdcTraderSpi::OnRspOrderAction(), if CTP local check failed;
  • CThostFtdcTraderSpi::OnErrRtnOrderAction(), if CTP local check passed but the request was rejected by brokerage(or exchange);
  • CThostFtdcTraderSpi::OnRtnOrder(), if brokerage(or exchange) accepted the request and the updated order status was returned.
Precondition
Login successfully.
See also
CThostFtdcTraderSpi::OnRspOrderAction(), CThostFtdcTraderSpi::OnErrRtnOrderAction(), CThostFtdcTraderApi::ReqOrderInsert()

◆ ReqOrderInsert()

virtual int CThostFtdcTraderApi::ReqOrderInsert ( CThostFtdcInputOrderField *  pInputOrder,
int  nRequestID 
)
pure virtual

The client application uses this function to send the order insertion request to the server.

Parameters
[in]pInputOrderA pointer to an CThostFtdcInputOrderField instance.
[in]nRequestIDThe request id.
Return values
0Success.
-1Network connection failure.
-2Exceeded the limit of max quantity of buffered(unhandled) requests.
-3Exceeded the limit of max requests per second.
Sample code
CThostFtdcInputOrderField field;
memset(&field, 0, sizeof(field));
strcpy(field.BrokerID, BROKER_ID);
strcpy(field.InvestorID, USER_ID);
strcpy(field.UserID, USER_ID);
strcpy(field.InstrumentID, "CU1909-SH");
// ...
strcpy(field.OrderRef, "1");
field.OrderPriceType = THOST_FTDC_OPT_LimitPrice;
field.Direction = THOST_FTDC_D_Buy;
field.CombOffsetFlag[0] = THOST_FTDC_OF_Open;
field.CombHedgeFlag[0] = THOST_FTDC_HF_Speculation;
field.CombHedgeFlag[1] = '9'; // set [1] to '9' for market maker order or '\0' for normal order.
field.CombHedgeFlag[2] = '2'; // set [2] to '2' for T+1 session flag, '1' for T session flag, '\0' for auto session flag(currently for HKEX only).
strcpy(field.BusinessUnit, "customized-data"); // Store customized data if necessary(needs to be a string)
field.LimitPrice = price; // Price
field.VolumeTotalOriginal = volume; // Volume
// ...
int rtnCode = tradeApi->ReqOrderInsert(&field, nextRequestID());
if (rtnCode != 0) {
Logger::info("[ERROR] [%s:%3d] Request failed: code=%d.", __FUNCTION__, __LINE__, rtnCode);
} else {
Logger::info("[INFO] [%s:%3d]: Requested to add new order: instrumentID=%s, direction=%c, volume=%d, price=%f.", __FUNCTION__, __LINE__,
field.InstrumentID, field.Direction, field.VolumeTotalOriginal, field.LimitPrice);
}
Note
When CTP server received the request, a local check will be held. If CTP check failed with the input parameters, CThostFtdcTraderSpi::OnRspOrderInsert() will be called with the error code and error message. If CTP check succeed, the request will be forwarded to the next hop(brokerage or exchange) and CThostFtdcTraderSpi::OnRspOrderInsert() won't be invoked in this case, the subsequent information will be notified through CThostFtdcTraderSpi::OnRtnOrder() and CThostFtdcTraderSpi::OnRtnTrade().
CThostFtdcInputOrderField.OrderRef is a string of an integer which should be increased each time and should keep unique during the session.
For non market maker order, user can use CThostFtdcInputOrderField.CombHedgeFlag[2] to set trading session flag. The T/T+1 session flag is currently available for order in HKEX and will be ignored for order in other exchanges. When there's status update for the order, the session flag received from exchanges will be returned via CThostFtdcOrderField.CombHedgeFlag[2].
User can use CThostFtdcInputOrderField.BusinessUnit field to store customized data. The data needs to be a null-terminated string. When there's status update or new trade for the order, the same BusinessUnit will be returned with CThostFtdcTraderSpi::OnRtnOrder() and CThostFtdcTraderSpi::OnRtnTrade() via CThostFtdcOrderField.BusinessUnit and CThostFtdcTradeField.BusinessUnit.
Precondition
Login successfully.
See also
CThostFtdcTraderSpi::OnRspOrderInsert(), CThostFtdcTraderSpi::OnRtnOrder(), CThostFtdcTraderSpi::OnRtnTrade()

This function is to query the information of instrument(s) with filters.

Multiple CThostFtdcInstrumentField structures maybe returned based on the input parameters.

Parameters

[in]nRequestIDThe request id.
Return values
0Success.

-3Exceeded the limit of max requests per second.
Precondition
Login successfully.
Sample code
CThostFtdcQryInstrumentField field;
memset(&field, 0, sizeof(field));
strcpy(field.InstrumentID, "");
strcpy(field.ExchangeID, "CME"); // Set other fields to empty to query all the CME instruments.
//strcpy(field.ExchangeInstID, "EVM"); // Enable this line to get CThostFtdcInstrumentField.VolumeMultiple with original currency.
strcpy(field.ProductID, "");
int rtnCode = tradeApi->ReqQryInstrument(&field, nextRequestID());
if (rtnCode != 0) {
Logger::info("[ERROR] [%s:%3d] Request failed: code=%d.", __FUNCTION__, __LINE__, rtnCode);

Logger::info("[INFO] [%s:%3d]: Requested to query instrument info: exchangeID=%s, instrumentID=%s, productID=%s.", __FUNCTION__, __LINE__,
field.ExchangeID, field.InstrumentID, field.ProductID);
}
Note
If no filter has been set, the full list of instrument will be returned.
User can get currency, rate and contract size from the returned CThostFtdcInstrumentField.ExchangeInstID.
The input parameter CThostFtdcQryInstrumentField.ExchangeInstID is reused as a flag, user can set this field to "EVM"(Exchange Volume Multiple) to get the lot size with returned CThostFtdcInstrumentField.VolumeMultiple, otherwise the returned CThostFtdcInstrumentField.VolumeMultiple will be convert to CNY based lot size.
See also
CThostFtdcTraderSpi::OnRspQryInstrument()

◆ SubscribePrivateTopic()

virtual void CThostFtdcTraderApi::SubscribePrivateTopic ( THOST_TE_RESUME_TYPE  nResumeType)
pure virtual

The client application uses this function to subscribe the trading notifications of the current trading day of the trading account, the notifications will be sent immediately when logon succeed via CThostFtdcTraderSpi::OnRtnOrder(), CThostFtdcTraderSpi::OnRtnTrade(), CThostFtdcTraderSpi::OnErrRtnOrderAction().

Parameters
[in]nResumeTypeThe retransmit mode of the private stream.
Mode Description
THOST_TERT_RESTART Retransmit from the begin of the current trading day. (This is the default value.)
THOST_TERT_RESUME Resume transmitting from the last received data.
THOST_TERT_QUICK Rransmitting the new private stream data from the login time.
Sample code
pTraderApi->SubscribePrivateTopic(THOST_TERT_RESTART);
Note
The function must be called before the invocation of CThostFtdcTraderApi::Init() to take effect.
